The journey begins
Published on: 04/04/2022 by Madalina

The first step I made with this website was to hand design the website mock, after that it was pretty easy to follow the mock and code the design.... Pretty easy right?

Well as everything that seems pretty easy, transcribing this to real life I got a big slap regarding my so tought "good coding skills".

At the start I made some huge mistakes because I read in between the brief lines, spent a lot of time designing and coding using flexbox (managed to do the header, footer, index and blog).

Luckily enough I read te brif again, and I noticed that we need to use the grid layout and changed all of them to grid layout wich if I may add i absolutely love as it's more easy to code with it.

Overall the most time-consuming and important part of the index page was the header and the footer as they repeat on every page and if I ruin the code there I could break the website design everywhere.

I found that making the branches for every part I want to code on the website made my life easier as I didn't have to stress over ruining a part of the code that was already good

Designing the portofolio

Published on: 11/04/2022 by Madalina

Because i transfered the initial blog code onto the portofolio page, I made a rookie mistake and didn't change the link from the blog.css to portofolio.css

This resulted in a few hours of running around and checking all over the internet why my changes didn't work on the website, after finally noticing that I forgot to change the link everything went on pretty smooth.

To resemble as a normal portofolio page, I left the design to be simple and pretty stright forward, I added some of my actual projects that I'm working on, linking them with the website url, so everyone can check them.

Unfortunately a lot of my past projects were closed by the owners and didn't have a workign link for them, but I still had screenshots of the website, so I converted them into a website mock image and displayed them on the page.

Overall I believe I added a few of them there I am personally happy with the way the page looks.

At the end I added CSS effects for the websites with the active link to shake and the ones witouth a link just to float to give the visual distinction of the link that it's working and the one that is not.

My experience from this project

Published on: 17/04/2022 by Madalina

Even tough in the beginig it felt I was going nowhere with the code and design, making plenty of mistakes and even crashing the website when I first merged the codes togheder.

This project really made me understand and create a logic of how things work in HTML and CSS, and working on it and keeping on making mistakes taught me that even though you know things theoretically, practice is the best way to learn how to do things.

In the early stages of building the website it felt like adding and removing things becomes more easy, even though I always have the constant fear that maybe I didn't do something properly, or maybe I forgot to add somethign in the back end.

As I was building websites for years using page builders because coding felt difficult to learn I can say that after this project I am more keen to design my own websites and not be constrainted by the plugins settings, ineffiiency.
